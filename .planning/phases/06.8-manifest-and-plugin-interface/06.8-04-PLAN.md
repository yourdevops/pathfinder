<objective>
Redesign the workflow creation and composer to derive runtime constraints from steps instead of requiring upfront selection.

Purpose: GAP-20 removes the artificial limitation of upfront runtime selection, enabling multi-runtime workflows and deriving version constraints from the intersection of included steps' requirements. This is the cornerstone of the CI Workflows model where workflows express what they need, not what they are.
Output: Updated model, forms, views, constraint intersection logic, composer UI with live constraint summary, and all template updates.
</objective>

  4. Add `intersect_semver_constraints()` pure function:
     - Takes a list of constraint strings (e.g., `[">=3.10", ">=3.11", "<4.0"]`)
     - Parses each into (operator, version) pairs using regex
     - Finds the highest lower bound (>=, >) and lowest upper bound (<=, <)
     - If lower > upper, constraints conflict (return None)
     - Otherwise return the tightest combined constraint string
     - Handle wildcards: filter out `"*"` before processing; if all are wildcards, return `"*"`
     - Use the existing `normalize_version()` helper and `semver.Version.parse()` for comparison

  5. Add `compute_runtime_constraints()` function:
     - Takes a list of CIStep instances (or dicts with `runtime_constraints` and `name`)
     - Collects all constraints per runtime family from each step
     - Steps with no runtime_constraints or `{"*": "*"}` are runtime-agnostic (don't restrict)
     - For each family, intersects all constraint values
     - Returns `{"constraints": {"python": ">=3.11", "node": ">=18"}, "conflicts": [...]}`
     - Conflicts include step names and the conflicting constraint values

  6. Add `check_step_constraint_compatibility()` function:
     - Takes current derived constraints dict and a candidate CIStep
     - Returns `{"compatible": True/False, "conflicts": [{"runtime": "python", "existing": ">=3.11,<3.12", "step_constraint": ">=3.13", "existing_steps": ["step-a"]}]}`
     - Used by the composer POST to validate before adding a step
