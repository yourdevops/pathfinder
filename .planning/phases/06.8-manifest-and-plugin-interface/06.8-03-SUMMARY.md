## Decisions Made
- Provisioning is non-blocking: CI variable provisioning failure does not prevent or roll back service creation
- GitHub variable API uses try/create-catch-409/update because create_variable is not idempotent (409 on existing)
- Static engine-native PTF variable references (${{ vars.PTF_PROJECT }}) are identical across all services, preserving manifest hash-based verification
