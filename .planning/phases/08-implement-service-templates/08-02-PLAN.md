**core/views/templates.py** — NEW file. Create views using OperatorRequiredMixin (from core/views/connections.py pattern):

1. `TemplateListView(LoginRequiredMixin, ListView)` — template_name `core/templates/list.html`, queryset `Template.objects.annotate(version_count=Count('versions'))`, context includes version count
2. `TemplateDetailView(LoginRequiredMixin, DetailView)` — lookup by `template_name` kwarg (dns path converter), shows template metadata + version list (tag + synced_at), sync status section
3. `TemplateRegisterView(OperatorRequiredMixin, FormView)` — handles registration flow:
   - GET: show TemplateRegisterForm
   - POST: shallow clone repo, read pathfinder.yaml via `read_pathfinder_manifest`, validate name uniqueness against Template model, create Template record, full clone to list semver tags and create TemplateVersion records, redirect to detail page
   - Use `build_authenticated_git_url` for private repos, `clone_repo_shallow`/`clone_repo_full` and `list_tags_from_repo`/`parse_version_tag` from git_utils
   - Handle errors (manifest not found, invalid manifest, name collision) with form error messages
   - Set `sync_status='synced'`, `last_synced_at=now`, populate `runtimes`, `required_vars`, `description` from manifest
4. `TemplateDeregisterView(OperatorRequiredMixin, DeleteView)` — hard delete with guard: if `template.services.exists()`, return 403/error message "Cannot delete template — services reference it". Otherwise delete (CASCADE will clean up TemplateVersion records). Redirect to list.
5. `TemplateSyncStatusView(LoginRequiredMixin, DetailView)` — HTMX partial returning just the sync status badge for auto-refresh during sync (same pattern as StepsRepoScanStatusView)

**core/views/__init__.py** — add imports for the new views
