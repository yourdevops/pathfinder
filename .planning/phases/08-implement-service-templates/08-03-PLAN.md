  <name>Task 1: sync_template background task</name>
  <files>core/tasks.py</files>
  <action>
Add a new `sync_template` task in core/tasks.py, following the `scan_steps_repository` pattern:

```python
@task(queue_name="steps_scan")  # reuse same queue
def sync_template(template_id: int) -> dict:
```

The task should:
1. Load Template record by ID. If not found, return error.
2. Mark `sync_status='syncing'`, clear `sync_error`.
3. Build authenticated URL if `template.connection` is set (use `build_authenticated_git_url`).
4. Full clone the repo (need full clone to list all tags): `clone_repo_full(template.git_url, auth_url=auth_url)`.
5. Get HEAD SHA. If same as `template.last_synced_sha`, mark synced and return early (skip optimization).
6. Read `pathfinder.yaml` from repo root via `read_pathfinder_manifest`. If missing or invalid, mark error and return.
7. Update Template metadata from manifest: `description`, `runtimes`, `required_vars`. Do NOT update `name` — it's immutable.
8. List all tags via `list_tags_from_repo(repo_obj)`. Filter to semver tags using `parse_version_tag` (skip non-semver).
9. For each semver tag:
   - If TemplateVersion with this tag_name exists for this template: skip (don't update)
   - If new: create TemplateVersion with `tag_name`, `commit_sha` (from tag info), `sort_key` (from parse_version_tag), `available=True`
10. Mark tags no longer present on remote: `TemplateVersion.objects.filter(template=template).exclude(tag_name__in=remote_tag_names).update(available=False)` — per locked decision: flag as unavailable, don't delete.
11. Update Template: `sync_status='synced'`, `last_synced_at=now`, `last_synced_sha=head_sha`.
12. Cleanup: `cleanup_repo(repo_obj, temp_dir)`.
13. Exception handling: on any error, set `sync_status='error'`, `sync_error=scrub_credentials(str(e))`.

Return dict with counts: `{"versions_added": N, "versions_unavailable": M}`.
  </action>

**Rewrite scaffold_repository** in core/tasks.py:
Replace the existing `scaffold_repository` task entirely (per locked decision: "Replace the existing scaffold_repository task entirely — it's a leftover from old Blueprints").

New flow:
1. Load Service (with select_related project, template) and IntegrationConnection.
2. Mark scaffold_status='running'.
3. Get template variables via `get_template_variables(service)`.
4. **If service.repo_is_new (new repo):**
   a. If service has a template and template_version:
      - Build auth URL for template repo clone
      - Full clone template repo: `clone_repo_full(template.git_url, auth_url=template_auth_url)`
      - Checkout the specific tag commit SHA: find TemplateVersion matching `service.template_version` tag_name, then `repo.git.checkout(version.commit_sha)`
      - Create the new service repo via plugin (existing `scaffold_new_repository` logic for repo creation + remote push)
      - Apply template files: `apply_template_to_directory(template_temp_dir, dest_dir, variables)` — pathfinder.yaml already in exclude list from Plan 01
      - If CI workflow assigned, generate manifest and include it
      - Commit and push to new repo
      - Cleanup template clone
   b. If no template (bare scaffolding):
      - Same as before: create empty repo, push README + optional CI manifest
5. **If existing repo:**
   - CI workflow manifest push remains separate mechanism (existing `push_ci_manifest` task) — per locked decision
   - Set scaffold_status='not_required' for existing repos
6. On success: scaffold_status='success', update repo_url.
7. On failure: scaffold_status='failed', scaffold_error with scrubbed credentials.
8. CI variable provisioning after scaffold (preserve existing behavior).

Key: The rewrite should keep the same function signature `scaffold_repository(service_id: int, scm_connection_id: int)` and same task decorator. The caller doesn't change.

In **core/git_utils.py**, if the `scaffold_new_repository` function passes `template_temp_dir=None` unconditionally, update it to accept a real template directory path. The function should:
- When `template_temp_dir` is provided and not None: copy template files using `apply_template_to_directory` with variable substitution
- When `template_temp_dir` is None: create a bare repo with just README.md (existing behavior)
