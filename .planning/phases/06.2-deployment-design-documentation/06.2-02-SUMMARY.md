## Decisions Made
- **Health Check as distinct state:** Documented `health_check` as a separate status in the state machine (not a sub-phase of "running") for meaningful UI feedback during deployment
- **Env var snapshot scope:** Full cascade (Project -> Environment -> Service) frozen at deployment creation time, stored on Deployment record for auditability
- **Rollback = re-deploy:** No special rollback action; user selects a previous successful build and promotes it, creating a fresh Deployment record
- **Ordering not enforced:** Environment ordering is recommended via UI indicators but not enforced, matching GitOps platform conventions (ArgoCD, Flux) rather than sequential enforcement (Spinnaker)
- **Concurrent blocking per pair:** Only (service, environment) pairs are blocked; different services in the same environment deploy independently
