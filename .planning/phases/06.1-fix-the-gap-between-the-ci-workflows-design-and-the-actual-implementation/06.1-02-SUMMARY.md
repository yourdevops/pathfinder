## Accomplishments
- verify_build task fetches manifest at build's commit SHA, computes SHA-256 hash, looks up CIWorkflowVersion, and sets verification_status (verified/draft/unauthorized)
- poll_build_details chains to verify_build on terminal builds (success/failed/cancelled)
- push_ci_manifest updated to use manifest_id(workflow) for file path, versioned manifest from pinned ci_workflow_version, and dual push mode (PR/direct)
- WorkflowComposerView.post() auto-creates/updates draft CIWorkflowVersion on every workflow save
- WorkflowDetailView exposes draft_version and latest_version in template context for Plan 03 UI
- WorkflowManifestView shows versioned manifest content with priority: draft > authorized > fresh

## Decisions Made

- push_ci_manifest does NOT add skip-CI flags to commits per locked decision: "No skip-CI on manifest commits"
- WorkflowManifestView shows draft manifest first, then latest authorized, then fresh generate as fallback
