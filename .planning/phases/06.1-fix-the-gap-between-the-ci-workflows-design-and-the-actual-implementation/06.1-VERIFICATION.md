- **Evidence:** Model fully implements versioning design per docs/ci-workflows/versioning.md

```python
    workflow=workflow,
    status=CIWorkflowVersion.Status.DRAFT,
    defaults={
        "manifest_hash": manifest_hash,
        "manifest_content": manifest_content,
        "author": request.user,
    },

```
- **Evidence:** Every workflow save in WorkflowComposerView.post() creates/updates draft version with regenerated manifest hash

### Truth 3: Build Verification Task
**Implementation (core/tasks.py:377-437):**
- 7-step verification flow per docs/ci-workflows/build-authorization.md
- Fetches manifest content at build's commit SHA via plugin.fetch_manifest_content
- Computes SHA-256 hash via compute_manifest_hash(content)
- Looks up CIWorkflowVersion by workflow + manifest_hash
- Sets verification_status: verified (authorized match), draft (draft match), unauthorized (no match or revoked)
- **Evidence:** Task implements complete verification flow with hash comparison

### Truth 4: Plugin Interface Extension
**Interface Declaration (plugins/base.py:124-144):**
- manifest_id(workflow) -> str
- extract_manifest_id(run_data) -> str | None
- get_manifest_id_pattern() -> re.Pattern
- fetch_manifest_content(config, repo_name, manifest_id, commit_sha) -> str | None

**GitHub Implementation (plugins/github/plugin.py):**
- manifest_id: returns f".github/workflows/ci-{workflow.name}.yml"
- extract_manifest_id: extracts from run_data["name"] if starts with "ci-"
- get_manifest_id_pattern: returns compiled regex for .github/workflows/ci-*.yml
- fetch_manifest_content: uses PyGithub get_contents(manifest_id, ref=commit_sha)
- **Evidence:** All 4 methods declared in base and implemented in GitHub plugin
