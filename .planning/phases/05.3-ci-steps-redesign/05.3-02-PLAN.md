  <name>Task 1: Refactor scan_steps_repository and push_ci_manifest tasks</name>
  <files>core/tasks.py, core/ci_manifest.py</files>
  <action>
  **Refactor `scan_steps_repository` in core/tasks.py:**

  1. Change imports: Replace `from core.git_utils import ... scan_ci_steps, parse_runtimes_yml` with `from core.ci_steps import discover_steps, parse_runtimes_yml`. Keep git_utils imports for `build_authenticated_git_url`, `cleanup_repo`, `clone_repo_shallow`.

  2. After cloning the repository, resolve the CI plugin for the repository's engine:
     ```python
     from plugins.base import get_ci_plugin_for_engine
     ci_plugin = get_ci_plugin_for_engine(repository.engine)
     if not ci_plugin:
         raise ValueError(f"No CI plugin installed for engine: {repository.engine}")
     ```

  3. Replace `steps_data = scan_ci_steps(temp_dir)` with:
     ```python
     discovered = discover_steps(temp_dir, ci_plugin.engine_file_name)
     steps_data = []
     for item in discovered:
         step_info = ci_plugin.parse_step_file(item['raw_content'])
         step_info['directory_name'] = item['directory_path']
         steps_data.append(step_info)
     ```

  4. In the CIStep.objects.update_or_create call, add `engine=repository.engine` to the defaults dict.

  **Refactor `push_ci_manifest` in core/tasks.py:**

  1. Replace `from core.ci_manifest import generate_github_actions_manifest` with `from plugins.base import get_ci_plugin_for_engine`.

  2. Remove the direct import of `parse_git_url` from core.git_utils (it's already imported elsewhere in the function -- just update as needed).

  3. After getting the service and validating the workflow, resolve CI plugin:
     ```python
     # Determine engine from workflow steps (first step's engine, or default)
     first_step = service.ci_workflow.workflow_steps.select_related('step').first()
     engine = first_step.step.engine if first_step else "github_actions"
     ci_plugin = get_ci_plugin_for_engine(engine)
     if not ci_plugin:
         return {"error": f"No CI plugin for engine: {engine}"}
     ```

  4. Replace `manifest_yaml = generate_github_actions_manifest(service.ci_workflow)` with:
     ```python
     manifest_yaml = ci_plugin.generate_manifest(service.ci_workflow)
     ```

  5. Replace hardcoded `manifest_path = f".github/workflows/{service.handler}.yml"` with:
     ```python
     manifest_path = ci_plugin.manifest_path(service)
     ```

  The SCM connection resolution stays the same (ProjectConnection lookup). This achieves the goal of resolving CI capability and SCM capability independently.

  **Clean core/ci_manifest.py:**

  Remove the `generate_github_actions_manifest` function entirely. Keep only:
  - `normalize_version`
  - `is_step_compatible`
  - `get_compatible_steps`

  Remove the `from core.models import CIStep` import at the top level -- move it to a lazy import inside `get_compatible_steps` if needed, or keep it since it's fine for this module. Actually, it's already a direct import and not circular, so keep it.

  Update the module docstring to reflect that it now only contains runtime compatibility checking, not manifest generation.
