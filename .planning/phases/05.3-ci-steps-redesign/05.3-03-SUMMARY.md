# Phase 5.3 Plan 03: Table Layouts, HTMX Filter Dropdowns Summary

Table-based UI for Steps Catalog and Repositories pages with HTMX-powered engine/runtime filtering

## What Was Done

### Task 1: Steps Catalog Table with HTMX Filters
- Replaced card-based phase-grouped layout with a 6-column table (Name, Phase, Engine, Runtimes, Tags, Repository)
- Added three inline filter dropdowns above the table: Engine, Runtime, Runtime Version
- Each dropdown triggers HTMX GET to `ci_workflows:steps_table` endpoint, updating only the `<tbody>` via partial rendering
- Created `_steps_table.html` as the HTMX partial for filtered row updates
- Clickable rows navigate to step detail page
- Empty state preserved when no steps exist
- Commit: `a5347e8`

### Task 2: Repositories Table, Engine Registration, Runtimes Deletion
- Replaced card layout with 5-column table (Name, Engine, URL, Steps, Last Synced)
- Uses `engine_display` annotation from view for human-readable engine names
- Added CI Engine dropdown to repository registration form between Name and Git URL fields
- Deleted `runtimes.html` template (RuntimesView was removed in Plan 02)
- Commit: `8285068`
