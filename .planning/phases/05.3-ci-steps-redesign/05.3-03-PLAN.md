     - Pre-select values from `selected_engine`, `selected_runtime`, `selected_runtime_version` context variables.
  3. Table with dark theme styling (consistent with existing tables in the app):
     ```html
     <table class="w-full">
         <thead>

             </tr>
         </thead>
         <tbody id="steps-table-body">
             {% include "core/ci_workflows/_steps_table.html" %}
         </tbody>
     </table>
     ```
  4. Empty state: if no steps, show a message.

  Use Tailwind dark theme classes consistent with the rest of the app: `bg-dark-bg`, `text-dark-text`, `border-dark-border`, `text-dark-muted`, etc.

  **Create `_steps_table.html`:**

  This is the HTMX partial that renders table rows:
  ```html
  {% for step in steps %}
  <tr class="border-b border-dark-border/50 hover:bg-dark-surface/50 cursor-pointer transition-colors"
      onclick="window.location.href='{% url 'ci_workflows:step_detail' step_uuid=step.uuid %}'">
      <td class="py-3 px-4">
          <span class="text-dark-text font-medium">{{ step.name }}</span>
          {% if step.description %}<p class="text-dark-muted text-xs mt-0.5">{{ step.description|truncatechars:60 }}</p>{% endif %}
      </td>
      <td class="py-3 px-4"><span class="text-xs px-2 py-0.5 rounded bg-dark-surface text-dark-muted">{{ step.get_phase_display|default:"—" }}</span></td>
      <td class="py-3 px-4 text-sm text-dark-muted">{{ step.engine_display_name|default:step.engine }}</td>
      <td class="py-3 px-4 text-sm text-dark-muted">
          {% for family, constraint in step.runtime_constraints.items %}{{ family }}{% if not forloop.last %}, {% endif %}{% empty %}—{% endfor %}
      </td>
      <td class="py-3 px-4">
          {% for tag in step.tags %}<span class="text-xs px-1.5 py-0.5 rounded bg-dark-border/50 text-dark-muted mr-1">{{ tag }}</span>{% empty %}—{% endfor %}
      </td>
      <td class="py-3 px-4 text-sm text-dark-muted">{{ step.repository.name }}</td>
  </tr>
  {% empty %}
  <tr><td colspan="6" class="py-8 text-center text-dark-muted">No steps match the selected filters.</td></tr>
  {% endfor %}
  ```

  Note: The `engine_display_name` may need to be annotated by the view. If the view passes a dict mapping engine names to display names, use a template filter or just display the engine field value. Keep it simple -- the raw `step.engine` value (e.g., `github_actions`) formatted nicely is acceptable: replace underscores with spaces and title-case, or the view can annotate each step with a display name. Simplest approach: just show `step.engine` and let the view add an `engine_display` attribute on each step object before passing to template.
