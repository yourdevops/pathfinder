## Decisions Made
- **URL normalization for matching**: Used parse_git_url to extract owner/repo from both webhook payload and stored git_url, then compared lowercase keys. This handles URL format differences (HTTPS vs SSH, trailing .git, etc.)
- **Optional signature verification**: If no webhook_secret configured on connection, signature check is skipped (not blocked). This allows webhook functionality even when secret is not set.
- **Concurrent scan prevention**: If repo.scan_status == "scanning", webhook silently returns 200 without enqueuing. Prevents task queue flooding from rapid pushes.
- **django_scheduled_tasks in INSTALLED_APPS**: Required because the package has its own models (ScheduledTaskRunLog) and migrations for tracking task execution history.
