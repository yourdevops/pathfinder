```python
def find_open_pr(self, config, repo_name, branch_name):
    """Find an open PR for the given branch name."""
    repo = self._get_repo(config, repo_name)
    owner = repo.owner.login
    pulls = repo.get_pulls(state='open', head=f'{owner}:{branch_name}')
    for pr in pulls:
        return {"number": pr.number, "html_url": pr.html_url, "title": pr.title}
    return None
```

**is_patch_bump helper (core/tasks.py):**

Create an `is_patch_bump` helper function at module level:

```python
def is_patch_bump(current_version_str: str, new_version_str: str) -> bool:
    """Check if new_version is a patch bump from current_version (same major.minor)."""
    import semver as semver_lib
    try:
        current = semver_lib.Version.parse(current_version_str)
        new = semver_lib.Version.parse(new_version_str)
        return (new.major == current.major
                and new.minor == current.minor
                and new > current)
    except ValueError:
        return False
```

**auto_update_services task (core/tasks.py):**

```python
@task(queue_name="default")
def auto_update_services(workflow_id: int, version_id: int) -> dict:
    """Auto-update services when a patch version is published."""
```

Logic:
1. Load the CIWorkflow and CIWorkflowVersion by ID. If either doesn't exist or version status != "authorized", return early.
2. Query services: `Service.objects.filter(ci_workflow_id=workflow_id, auto_update_patch=True, ci_workflow_version__isnull=False)` -- only services that have a version pinned and auto-update enabled.
3. For each service, check `is_patch_bump(service.ci_workflow_version.version, new_version.version)`. Skip if not a patch bump.
4. For each eligible service:
   a. Update `service.ci_workflow_version = new_version` (FK update -- immediate)
   b. Set `service.ci_manifest_status = "pending_pr"`
   c. Save the service
   d. Enqueue `push_ci_manifest.enqueue(service_id=service.id)`
5. Return `{"updated": count, "skipped": skipped_count}`.

Note: Services are iterated and push_ci_manifest is enqueued for each. Concurrent execution for services in the same repo is safe because push_ci_manifest's digest PR detection checks for existing open PRs before creating a new branch -- both tasks will find the PR and add commits.
