# Phase 5.1 Plan 01: Blueprint Removal and CI Workflow Models Summary

**One-liner:** Removed Blueprint/BlueprintVersion models, added 5 CI Workflow domain models, cleaned all blueprint references, updated sidebar to CI Workflows.

## What Was Done

### Task 1: Remove Blueprint models, update Service model, run migrations
- **Commit:** `7cbb317`
- Removed `Blueprint` and `BlueprintVersion` classes from `core/models.py`
- Removed `blueprint` and `blueprint_version` ForeignKey fields from `Service` model
- Cleared `core/views/blueprints.py` (empty module for repurposing in Plan 02)
- Removed all Blueprint view imports from `views/__init__.py`
- Replaced `blueprints_patterns` with `ci_workflows_patterns` in `core/urls.py`
- Updated `devssp/urls.py`: `blueprints/` path to `ci-workflows/` with `ci_workflows` namespace
- Removed `sync_blueprint` task from `tasks.py`
- Updated `scaffold_repository` task to remove blueprint references
- Applied migration 0010 to drop Blueprint tables and Service FK columns

### Task 2: Clean blueprint references from views, forms, templates
- **Commit:** `bcf34d3`
- Replaced `BlueprintStepForm` with `ProjectStepForm` (project + service name selection)
- Updated `ServiceCreateWizard` to 4-step flow: project, repository, configuration, review
- Removed `BlueprintVersionsView` HTMX endpoint
- Updated `ServiceListView` queryset to `select_related('project')` only
- Created `step_project.html` template (replaces `step_blueprint.html`)
- Cleaned all blueprint columns from service list, services tab, details tab, review step
- Deleted entire `core/templates/core/blueprints/` directory (5 templates)
- Deleted `core/templates/core/placeholders/blueprints.html`
- Updated dashboard hero text and git_utils scaffold commit messages
- Updated Operator role description in forms

### Task 3: Add CI Workflow models and update navigation
- **Commit:** `306d699`
- Added `StepsRepository` model (git repos with CI step definitions)
- Added `RuntimeFamily` model (python, node, go families with version lists)
- Added `CIStep` model (individual steps from action.yml files)
- Added `CIWorkflow` model (composed workflow of ordered steps)
- Added `CIWorkflowStep` model (step-within-workflow join table)
- Registered all new models with auditlog
- Replaced Blueprints sidebar section with CI Workflows (Workflows, Steps, Runtimes, Repositories)
- Moved Resources to standalone nav item
- Applied migration 0011 creating 5 new tables

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Forms/views had Blueprint imports blocking migrations**
- **Found during:** Task 1
- **Issue:** `core/forms/services.py` and `core/views/services.py` imported `Blueprint`/`BlueprintVersion`, preventing `manage.py makemigrations` from running
- **Fix:** Updated forms and views imports as part of Task 1 (originally planned for Task 2) to unblock migrations
- **Files:** `core/forms/services.py`, `core/forms/__init__.py`, `core/views/services.py`

**2. [Rule 3 - Blocking] SQLite migration ordering issue**
- **Found during:** Task 1
- **Issue:** Django auto-generated migration tried to remove FK from BlueprintVersion before deleting the model, causing SQLite error
- **Fix:** Wrote custom migration (0010) with correct operation order: remove Service FKs first, then delete BlueprintVersion, then Blueprint
- **Files:** `core/migrations/0010_remove_blueprints_and_service_fks.py`

**3. [Rule 1 - Bug] Additional blueprint references in tasks.py, git_utils.py, projects.py, forms/base.py**
- **Found during:** Task 2
- **Issue:** Several files outside the plan's file list had Blueprint references that would cause runtime errors
- **Fix:** Cleaned all references: removed sync_blueprint task, updated scaffold_repository, fixed select_related in projects.py, updated Operator role description, cleaned git_utils commit messages
- **Files:** `core/tasks.py`, `core/git_utils.py`, `core/views/projects.py`, `core/forms/base.py`

**4. [Rule 1 - Bug] Old blueprint template directory still existed**
- **Found during:** Task 2
- **Issue:** `core/templates/core/blueprints/` directory with 5 templates (list, detail, register, preview, sync_status) was not in the plan's files list but contained dead references
- **Fix:** Deleted entire directory
- **Files:** `core/templates/core/blueprints/*`
