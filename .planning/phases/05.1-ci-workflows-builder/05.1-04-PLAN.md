  <action>
    **In core/ci_manifest.py, add:**

    `generate_github_actions_manifest(workflow) -> str`:
    - `workflow` is a CIWorkflow model instance
    - Build the manifest dict preserving insertion order (Python 3.7+ dict):
      ```python
      manifest = {
          'name': f'CI - {workflow.name}',
          'on': {
              'push': {'branches': ['main']},
          },
          'jobs': {
              'build': {
                  'runs-on': 'ubuntu-latest',
                  'steps': [],
              },
          },
      }
      ```
    - Add auto-injected steps (per docs/ci-workflows.md):
      1. Checkout: `{'name': 'Checkout', 'uses': 'actions/checkout@v4'}`
      2. SSP Notify Start: `{'name': 'Notify SSP - Build Started', 'uses': './ci-steps/ssp-notify-start'}`
    - Add user-composed steps from `workflow.workflow_steps.order_by('order')`:
      - For each CIWorkflowStep `ws`:
        - `uses`: format as `{repo_owner}/{repo_name}/ci-steps/{step_dir_name}@{commit_sha}` if repo git_url is parseable, otherwise `./{repo_name}/ci-steps/{step_dir_name}`
        - Use `parse_git_url()` from git_utils to extract owner/repo from the step repository's git_url
        - If `ws.input_config` is non-empty, add `with:` block
        - Step name from `ws.step.name`
    - Add auto-injected SSP Notify Complete: `{'name': 'Notify SSP - Build Complete', 'uses': './ci-steps/ssp-notify-complete'}`
    - Return `yaml.dump(manifest, default_flow_style=False, sort_keys=False)`
    - **PITFALL**: Always use `sort_keys=False` to preserve conventional YAML key ordering

    **Add views to core/views/ci_workflows.py:**

    1. `WorkflowListView(LoginRequiredMixin, View)`:
       - GET: List all CIWorkflow records, ordered by name

       - Include "+ Create CI Workflow" button
       - Template: `workflow_list.html`

    2. `WorkflowDetailView(LoginRequiredMixin, View)`:
       - GET: Show workflow details
       - Lookup by `dns:workflow_name`
       - Context: workflow, workflow_steps (ordered), manifest_yaml (from generate_github_actions_manifest)
       - Template: `workflow_detail.html`

    3. `WorkflowManifestView(LoginRequiredMixin, View)`:
       - GET HTMX endpoint: Return `_manifest_preview.html` partial with generated YAML
       - Useful for refreshing manifest preview without page reload

    4. `WorkflowDeleteView(OperatorRequiredMixin, View)`:
       - POST: Delete workflow if no services reference it (for now, just delete since no Service->Workflow FK yet)
       - Redirect to workflow list

    **Update core/urls.py ci_workflows_patterns:**

    ```python
    path('<dns:workflow_name>/delete/', WorkflowDeleteView.as_view(), name='workflow_delete'),
    ```
    **IMPORTANT:** Place the empty-path `workflow_list` at the START of ci_workflows_patterns. Place the `<dns:workflow_name>/` patterns AFTER all fixed-path patterns (repos/, steps/, runtimes/, create/, etc.) to avoid URL conflicts.
