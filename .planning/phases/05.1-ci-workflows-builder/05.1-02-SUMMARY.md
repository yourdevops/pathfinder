# Phase 5.1 Plan 02: Repository Scanning and Steps Catalog Summary

**One-liner:** Built repository scanning pipeline with parse_runtimes_yml/scan_ci_steps helpers, background scan task, 8 views, and 7 templates for steps catalog browsing.

## What Was Done

### Task 1: Repository scanning helpers and background task
- **Commit:** `5b09003`
- Added `parse_runtimes_yml()` to `core/git_utils.py` -- parses runtimes.yml supporting both `{family: {versions: [...]}}` and `{family: [...]}` formats, converts all versions to strings
- Added `scan_ci_steps()` to `core/git_utils.py` -- walks `ci-steps/` directory, parses action.yml (with action.yaml fallback), extracts x-pathfinder metadata (phase, runtimes, tags, produces)
- Added `scan_steps_repository` background task to `core/tasks.py` -- clones repo, parses runtimes and steps, creates/updates RuntimeFamily and CIStep records, deletes removed entries, handles errors with scan_error field
- Added `steps_scan` queue to `pathfinder/settings.py` TASKS QUEUES configuration

### Task 2: Repository and Steps Catalog views, forms, URLs, templates
- **Commit:** `0992e21`
- Created `core/forms/ci_workflows.py` with `StepsRepoRegisterForm` (name with dns_label_validator, git_url, optional connection dropdown filtered to active GitHub connections)
- Created `core/views/ci_workflows.py` with 8 views:
  - `StepsRepoListView` -- card grid with scan status badges, step/runtime counts
  - `StepsRepoRegisterView` -- form with name/url/connection, enqueues scan on submit
  - `StepsRepoDetailView` -- repo info, runtimes list, steps grouped by phase
  - `StepsRepoScanView` -- POST trigger for rescan with HTMX support
  - `StepsRepoScanStatusView` -- HTMX polling partial (3s interval while scanning)
  - `StepsCatalogView` -- all steps grouped by phase (setup/build/test/package)
  - `StepDetailView` -- full step metadata with inputs schema, runtime constraints, tags
  - `RuntimesView` -- runtime families grouped by repository
- Wired `ci_workflows_patterns` in `core/urls.py` with 8 URL routes
- Created 7 templates in `core/templates/core/ci_workflows/` with dark theme
- Updated sidebar nav.html -- Steps, Runtimes, Repositories links now use live URLs (Workflows stays as `#` for Plan 03)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] steps_scan queue not registered in TASKS config**
- **Found during:** Task 1
- **Issue:** django-tasks validates queue names at module import time; `@task(queue_name='steps_scan')` caused `InvalidTaskError` because `steps_scan` was not in QUEUES list
- **Fix:** Added `steps_scan` to `pathfinder/settings.py` TASKS QUEUES configuration
- **Files:** `pathfinder/settings.py`
- **Commit:** `5b09003`
