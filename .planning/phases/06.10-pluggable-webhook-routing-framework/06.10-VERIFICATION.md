### Summary

Phase 6.10 fully achieves its goal. All GitHub-specific webhook code has been removed from `core/` and relocated to `plugins/github/webhooks.py`. The single endpoint `/integrations/github/webhook/` is registered through the existing plugin URL autodiscovery mechanism and dispatches by `X-GitHub-Event` header. The `BasePlugin.get_webhook_url()` interface enables any future plugin to expose a webhook endpoint without any changes to core code. All three core webhook URL consumers (`ServiceRegisterWebhookView`, `push_ci_manifest`, `StepsRepoRegisterView`) and the GitHub App manifest builder use plugin-agnostic URL resolution. Zero hardcoded `/webhooks/` URL strings remain in any Python source file.

Architecture Rule 1 (core must be plugin-agnostic) is fully enforced for webhook handling.
