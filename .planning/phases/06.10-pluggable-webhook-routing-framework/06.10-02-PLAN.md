In `plugins/github/views.py`, update the `_build_manifest()` method (around line 157):

Replace the fragile string manipulation:
```python
# Webhook URL uses the same mount point; no route defined yet
webhook_path = callback_path.rsplit("manifest/callback/", 1)[0] + "webhook/"
```

With robust reverse lookup:
```python
from django.urls import reverse
webhook_path = reverse("github:webhook")
```

This is safe because `reverse("github:webhook")` returns `/integrations/github/webhook/` which is the correct path. The `hook_attributes.url` in the manifest will be `{external_url}/integrations/github/webhook/`.
