# Phase 5.2 Plan 02: Service CI Tab & Manifest Push Summary

**One-liner:** Service CI tab with workflow assignment dropdown, manifest sync status badges, push-to-repo button, and YAML preview; push_ci_manifest task creates feature branch + PR via GitHub API.

## What Was Done

### Task 1: Add Service CI tab with workflow assignment and manifest preview
- Extended ServiceDetailView with "ci" tab support including workflow, status, and manifest context
- Created ServiceAssignWorkflowView (POST) for workflow assignment with project-approved list validation
- Created ServicePushManifestView (POST) with guards for workflow, repo_url, and SCM connection
- Created `_ci_tab.html` template with four sections:
  - Assigned Workflow: dropdown of project-approved workflows with Save button
  - Manifest Status: badges for never_pushed (gray), synced (green), out_of_date (amber)
  - Push to Repository: button to trigger manifest push (disabled without workflow)
  - Manifest Preview: generated YAML in monospace code block
- Added CI Workflow nav item to nav_service.html between Details and Builds
- Added URL patterns for assign-workflow and push-manifest endpoints

### Task 2: Implement push_ci_manifest task and GitHubPlugin update_or_create_file
- Added `update_or_create_file` method to GitHubPlugin that handles both new and existing files
- Created `push_ci_manifest` background task on `repository_scaffolding` queue:
  - Generates manifest YAML from assigned workflow
  - Resolves default SCM connection from project
  - Creates feature branch `pathfinder/ci-manifest-{service.name}` (handles existing branch)
  - Writes `.github/workflows/{handler}.yml` via update_or_create_file
  - Opens PR with descriptive title and body
  - Updates service status fields (synced, pushed_at, pr_url)
