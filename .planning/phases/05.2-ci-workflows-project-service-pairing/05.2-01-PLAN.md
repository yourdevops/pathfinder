<task type="auto">
  <name>Task 1: Add CI pairing models and Service CI fields</name>
  <files>core/models.py</files>
  <action>
Add the following to core/models.py:

1. **CIWorkflow status field**: Add `status = models.CharField(max_length=20, choices=[("published", "Published"), ("draft", "Draft")], default="published")` to the CIWorkflow model. This future-proofs for draft support while treating all existing workflows as published.

2. **ProjectApprovedWorkflow model** (M2M through table):
   - `project` FK to Project (on_delete=CASCADE, related_name="approved_workflows")
   - `workflow` FK to CIWorkflow (on_delete=CASCADE, related_name="project_approvals")
   - `approved_by` CharField(max_length=150, blank=True) - denormalized username
   - `created_at` DateTimeField(auto_now_add=True)
   - `unique_together: ["project", "workflow"]`
   - `db_table = "core_project_approved_workflow"`

3. **ProjectCIConfig model** (OneToOne per project):
   - `project` OneToOneField to Project (on_delete=CASCADE, related_name="ci_config")
   - `default_workflow` FK to CIWorkflow (on_delete=SET_NULL, null=True, blank=True, related_name="default_for_projects")
   - `approve_all_published` BooleanField(default=False)
   - `created_at` DateTimeField(auto_now_add=True)
   - `updated_at` DateTimeField(auto_now=True)
   - `db_table = "core_project_ci_config"`

4. **Service model additions**: Add these fields to the existing Service model:
   - `ci_workflow` FK to CIWorkflow (on_delete=SET_NULL, null=True, blank=True, related_name="services")
   - `ci_manifest_status` CharField(max_length=20, choices=[("never_pushed", "Never Pushed"), ("synced", "Synced"), ("out_of_date", "Out of Date")], default="never_pushed")
   - `ci_manifest_pushed_at` DateTimeField(null=True, blank=True)
   - `ci_manifest_pr_url` URLField(max_length=500, blank=True)

5. **Add a helper method on Service**:
   ```python
   @property
   def ci_manifest_out_of_date(self):
       """Check if the manifest needs re-pushing."""
       if not self.ci_workflow or not self.ci_manifest_pushed_at:
           return False
       return self.ci_workflow.updated_at > self.ci_manifest_pushed_at
   ```

6. **Add a helper function** (can be a standalone function or on Project model):
   ```python
   def get_available_workflows_for_project(project):
       """Return queryset of CI workflows available for a project."""
       from core.models import CIWorkflow, ProjectCIConfig
       try:
           ci_config = project.ci_config
       except ProjectCIConfig.DoesNotExist:
           ci_config = None
       if ci_config and ci_config.approve_all_published:
           return CIWorkflow.objects.filter(status="published")
       approved_ids = project.approved_workflows.values_list("workflow_id", flat=True)
       return CIWorkflow.objects.filter(id__in=approved_ids, status="published")
   ```

7. **Register new models with auditlog**: Add `auditlog.register(ProjectApprovedWorkflow)` and `auditlog.register(ProjectCIConfig)` alongside existing auditlog registrations at the bottom of models.py.

8. **Run makemigrations and migrate**: `uv run python manage.py makemigrations core` and `uv run python manage.py migrate`.
