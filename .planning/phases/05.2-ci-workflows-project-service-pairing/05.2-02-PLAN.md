1. **Add `update_or_create_file` method to GitHubPlugin** in `plugins/github/plugin.py`:
   ```python
   def update_or_create_file(self, config, repo_name, path, content, message, branch="main"):
       """Create or update a file in a GitHub repository.

       Handles both cases: creates the file if it doesn't exist,
       updates it (with the current SHA) if it does.
       """
       g = self._get_github_client(config)
       repo = g.get_repo(repo_name)
       try:
           existing = repo.get_contents(path, ref=branch)
           result = repo.update_file(path, message, content, existing.sha, branch=branch)
       except Exception:
           result = repo.create_file(path, message, content, branch=branch)
       return {
           "path": result["content"].path,
           "sha": result["content"].sha,
           "commit_sha": result["commit"].sha,
       }
   ```

2. **Add `push_ci_manifest` task** in `core/tasks.py`:
   Use the `@task(queue_name="repository_scaffolding")` decorator. The task:

   a. Loads the Service with select_related("project", "ci_workflow")
   b. Validates ci_workflow is assigned (return error if not)
   c. Generates manifest YAML via `generate_github_actions_manifest(workflow)`
   d. Resolves SCM connection: `ProjectConnection.objects.filter(project=service.project, is_default=True).select_related("connection").first()`. If no connection found, log error and update service status.
   e. Gets the plugin and config from the connection
   f. Parses repo URL via `parse_git_url(service.repo_url)` to get owner/repo
   g. Creates branch: `pathfinder/ci-manifest-{service.name}` from `service.repo_branch` (or "main"). Use try/except to handle branch already existing (force-update scenario -- if branch exists, the update_or_create_file will still work on the existing branch).
   h. Writes manifest file: `.github/workflows/{service.handler}.yml` using `plugin.update_or_create_file()`
   i. Creates PR via `plugin.create_pull_request()` with title "Update CI workflow for {service.name}" and body describing the workflow name
   j. Updates service: `ci_manifest_status="synced"`, `ci_manifest_pushed_at=timezone.now()`, `ci_manifest_pr_url=pr_url`
   k. Returns `{"status": "success", "pr_url": pr_url}`
   l. On any exception: log the error, set `ci_manifest_status` to the previous value or leave as-is, return error dict

   **Important**: Use lazy imports inside the task function body (pattern already used in existing tasks) to avoid circular imports.

   **Branch handling**: The `create_branch` method may fail if branch already exists. Wrap it in try/except and continue -- the file update will still work on the existing branch. If a previous PR is still open, the new commit will be added to it.
