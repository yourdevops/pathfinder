1. **Create WorkflowSelectionForm** in `core/forms/services.py`:
   - `ci_workflow` ModelChoiceField with `queryset=CIWorkflow.objects.none()` (populated dynamically), `required=False`, `empty_label="No CI Workflow"`.
   - Style the select widget with dark theme classes matching other wizard forms: `class="w-full px-3 py-2 bg-dark-bg border border-dark-border rounded-lg text-dark-text"`.
   - Add `__init__` that accepts `project` kwarg. When project is provided:
     - Use `get_available_workflows_for_project(project)` to set the queryset
     - Pre-select the project's default workflow via `self.fields["ci_workflow"].initial = ci_config.default_workflow` if ci_config exists
   - Add help_text: "Select a CI Workflow to assign to this service. Only workflows approved for the project are shown."

2. **Update WIZARD_FORMS** in `core/views/services.py`:
   Insert `("workflow", WorkflowSelectionForm)` between `("configuration", ...)` and `("review", ...)`:
   ```python
   WIZARD_FORMS = [
       ("project", ProjectStepForm),
       ("repository", RepositoryStepForm),
       ("configuration", ConfigurationStepForm),
       ("workflow", WorkflowSelectionForm),
       ("review", ReviewStepForm),
   ]
   ```

3. **Update WIZARD_TEMPLATES**: Add `"workflow": "core/services/wizard/step_workflow.html"`.

4. **Update STEP_TITLES**: Add `"workflow": "CI Workflow"`.

5. **Update wizard docstring**: Change "4-step" to "5-step" and add "workflow" to the step list.

6. **Update imports** in services.py: Add `WorkflowSelectionForm` to the imports from `core.forms.services`.

7. **Update `get_form_kwargs()`** in ServiceCreateWizard: For the "workflow" step, pass the selected project as a kwarg. Get the project from the cleaned data of the "project" step:
   ```python
   if step == "workflow":
       project_data = self.get_cleaned_data_for_step("project")
       if project_data:
           kwargs["project"] = project_data.get("project")
   ```
   If `get_form_kwargs` doesn't exist yet, create it. If it does, add the workflow step handling.

8. **Update `get_context_data()`**: Add workflow step context (step title, any helper info).

9. **Update `done()` method**: After creating the service, set `service.ci_workflow` from the workflow step cleaned data:
   ```python
   workflow_data = self.get_cleaned_data_for_step("workflow")
   if workflow_data and workflow_data.get("ci_workflow"):
       service.ci_workflow = workflow_data["ci_workflow"]
       service.save(update_fields=["ci_workflow"])
   ```

10. **Create `step_workflow.html`** template: Follow the pattern of existing wizard step templates (extend base.html). Show:
    - Step description: "Select a CI Workflow for this service"
    - The ci_workflow dropdown field
    - Helper text about what CI workflows do
    - "Skip" note explaining that workflow can be assigned later from the service CI tab

11. **Update `step_review.html`**: Add a "CI Workflow" row to the review summary showing the selected workflow name (or "None" if skipped). Place it after the Configuration section.
