### Human Verification Required

#### 1. Copy Button Behavior

**Test:** Open the workflow composer, add a step with declared outputs (e.g., a packaging step), expand its Configure panel, click the Outputs tab, and click the copy icon next to an output.
**Expected:** The clipboard receives the engine-native reference string (e.g., `${{ steps.package-docker.outputs.image-ref }}`). The icon changes to a checkmark briefly.
**Why human:** Clipboard interaction and icon state change require browser interaction.

#### 2. Real-Time Inline Validation UX

**Test:** In the composer, add two steps where step A has outputs. In step B's Inputs tab, type or paste a reference like `${{ steps.a.outputs.nonexistent }}` into an input field.
**Expected:** A red error message appears immediately below the input field saying "Output 'nonexistent' not found on step 'a'".
**Why human:** Visual rendering of inline error and timing of Alpine.js reactive update requires browser interaction.

#### 3. Reorder Re-Validation

**Test:** Set up a workflow with step A (has output `image-ref`) and step B (input referencing `${{ steps.a.outputs.image-ref }}`). Drag step B before step A.
**Expected:** After reorder, step B's input field shows a red inline error: "Step 'a' must appear before this step".
**Why human:** Drag-and-drop interaction and reactive re-validation requires browser testing.

#### 4. Server-Side Rejection on Save

**Test:** Manually craft a POST with an invalid output reference (or use the UI with a broken ref) and submit the workflow.
**Expected:** The page re-renders with a Django message error listing the invalid reference. The workflow is not saved.
**Why human:** Requires a full browser request cycle; error message display is visual.
