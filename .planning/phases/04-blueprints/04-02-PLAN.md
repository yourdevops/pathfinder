  <action>
Create core/views/blueprints.py with the following views:

**BlueprintListView(LoginRequiredMixin, View):**
- template_name = 'core/blueprints/list.html'
- GET: Query Blueprint.objects.exclude(sync_status='pending').select_related('connection')
- Collect all_tags (unique tags from all blueprints) and deploy_plugins for filter dropdowns
- Get available_plugins from active IntegrationConnections
- Build blueprint_data list with {'blueprint': bp, 'is_available': bp.is_available_globally(), 'required_plugin': bp.deploy_plugin if not available else None}
- Check can_manage = has_system_role(request.user, ['admin', 'operator'])
- Render with blueprints, all_tags, deploy_plugins, can_manage

**BlueprintPreviewView(OperatorRequiredMixin, View):**
- HTMX endpoint for fetching and previewing manifest before registration
- POST only (accepts git_url and connection_id from form)
- Import from core.git_utils: parse_git_url, clone_repo_shallow, read_manifest_from_repo, cleanup_repo, build_authenticated_git_url
- Parse git_url with parse_git_url() - return error partial if invalid format
- Determine authentication:
  - If connection_id is provided and not empty: get IntegrationConnection
  - If connection_id is empty/None: connection = None (public access)
- Build authenticated Git URL using build_authenticated_git_url(git_url, connection)
- Clone repository using GitPython (not GitHub API):
  - repo, temp_dir = clone_repo_shallow(git_url, 'main', auth_url)
- Read manifest from cloned repo:
  - manifest = read_manifest_from_repo(temp_dir)
- Clean up temp directory: cleanup_repo(repo, temp_dir)
- Return _preview.html partial with:
  - preview_valid: True if manifest parsed successfully
  - preview_data: dict with name, description, tags, ci_plugin, deploy_plugin
  - preview_error: error message if failed (missing file, invalid YAML, clone error, etc.)
- On any exception: clean up temp dir, return _preview.html with preview_valid=False and preview_error

IMPORTANT: Use GitPython for Git operations (not GitHub API or requests.get to raw URLs). This enables SCM-agnostic preview that works with GitHub, GitLab, Bitbucket, or any Git server.

**BlueprintRegisterView(OperatorRequiredMixin, View):**
- template_name = 'core/blueprints/register.html'
- GET:
  - Query active GitHub connections: IntegrationConnection.objects.filter(plugin_type='github', is_active=True)
  - Render form with github_connections list (for dropdown)
- POST:
  - Get git_url from request.POST, strip whitespace
  - Get connection_id from request.POST (can be empty string for public repos)
  - Validate git_url not empty
  - Validate URL format with parse_github_url
  - Check for duplicate git_url
  - **CRITICAL: Require preview was successful** - re-fetch manifest to validate (or use session cache)
  - Create Blueprint with:
    - git_url=git_url
    - connection=IntegrationConnection.objects.get(id=connection_id) if connection_id else None
    - sync_status='pending'
    - created_by=request.user.username
  - Enqueue sync_blueprint task
  - Redirect to blueprints:detail (not list) with success message per CONTEXT.md

**BlueprintDetailView(LoginRequiredMixin, View):**
- template_name = 'core/blueprints/detail.html'
- GET: get_object_or_404(Blueprint, uuid=uuid)
- Get show_prereleases from request.GET.get('show_prereleases') == 'true'
- Filter versions (exclude prereleases unless show_prereleases)
- Get selected_version from request.GET.get('version') or use blueprint.latest_version
- Check is_available = blueprint.is_available_globally()
- Check can_manage = has_system_role(request.user, ['admin', 'operator'])
- Render with blueprint, versions, current_version, show_prereleases, prerelease_count, is_available, required_plugin, can_manage

**BlueprintSyncView(OperatorRequiredMixin, View):**
- POST only: get_object_or_404(Blueprint, uuid=uuid)
- If sync_status != 'syncing': mark as syncing, enqueue sync_blueprint task
- If HTMX request: return partial _sync_status.html
- Otherwise: redirect to blueprints:detail

Update core/views/__init__.py to import and export:
- BlueprintListView (as BlueprintsListView for backwards compat with URL)
- BlueprintDetailView
- BlueprintRegisterView
- BlueprintPreviewView
- BlueprintSyncView
