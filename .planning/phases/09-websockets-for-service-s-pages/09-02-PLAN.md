  <action>
    Extract three sections from `_details_tab.html` into separate partial templates. Each partial MUST own its outer wrapping div with a stable `id` attribute. This div is the OOB swap target. The partial also conditionally adds the `hx-swap-oob="true"` attribute when rendered with `oob=True` in the template context. This avoids double-wrapping: when included via `{% include %}` on initial page load, `oob` is not set so the div is a plain container; when rendered by the WebSocket consumer with `oob=True`, the div becomes the OOB swap element.

    **OOB pattern for every partial's root element:**
    ```html
    <div id="dashboard-stats-row" {% if oob %}hx-swap-oob="true"{% endif %}>
        ...content...
    </div>
    ```

    1. **Create `_stats_row.html`** -- extract Section 2 (Stats Row) from `_details_tab.html`:
       - Root element: `<div id="dashboard-stats-row" {% if oob %}hx-swap-oob="true"{% endif %}>`
       - Contains: the 4-card grid (Last Build, Success Rate, Total Builds, Avg Build Time)
       - Make "Total Builds" card clickable -- wrap in `<a>` linking to builds tab:
         `href="{% url 'projects:service_detail' project_name=project.name service_name=service.name %}?tab=builds"`
         with `hx-get` + `hx-target="#tab-content"` + `hx-push-url="true"` matching existing pattern
       - Make "Last Build" card clickable -- same link pattern to builds tab
       - Keep Success Rate and Avg Build Time as non-clickable (no meaningful drill-down target)
       - Requires template context: `last_build`, `success_rate`, `total_builds`, `avg_build_time_seconds`, `project`, `service`

    2. **Create `_recent_builds.html`** -- extract Section 4 (Recent Builds) from `_details_tab.html`:
       - Root element: `<div id="dashboard-recent-builds" {% if oob %}hx-swap-oob="true"{% endif %}>`
       - Make commit SHAs clickable: wrap `{{ build.commit_sha|slice:":7" }}` in `<a>` tag
         - If `build.ci_run_url` exists, link to it (links to the CI run page which shows the commit)
         - If service has `repo_url`, construct commit URL: `{{ service.repo_url|strip_git_suffix }}/commit/{{ build.commit_sha }}`
         - Fallback: render as plain text if neither URL is available
       - Requires template context: `recent_builds`, `total_builds`, `project`, `service`

    3. **Create `_ci_pipeline_card.html`** -- extract Section 3 (CI Pipeline Status) from `_details_tab.html`:
       - Root element: `<div id="dashboard-ci-pipeline" {% if oob %}hx-swap-oob="true"{% endif %}>`
       - Add left accent border to the card based on CI pipeline health:
         - Green `border-l-4 border-l-green-500`: ci_workflow assigned AND ci_manifest_status == "synced" AND webhook_registered AND last_build exists AND last_build.status == "success"
         - Amber `border-l-4 border-l-amber-500`: ci_workflow assigned AND (ci_manifest_status is "out_of_sync" or last_build.status == "failed" or not webhook_registered)
         - No accent (just the standard border): no ci_workflow assigned or never_pushed
       - For CI Pipeline empty state (no workflow assigned): use dashed-border card with pipeline icon instead of plain centered text:
         ```html
         <div class="border-2 border-dashed border-dark-border rounded-lg p-6 text-center">
             <svg ...pipeline icon...>
             <p class="text-dark-muted text-sm mt-2">No CI workflow assigned</p>
             <a ...link to CI tab...>Assign a workflow</a>
         </div>
         ```
       - Requires template context: `ci_workflow`, `ci_workflow_version`, `service`, `project`, `last_build`

    4. **Refactor `_details_tab.html`**:
       - Replace extracted sections with `{% include %}` calls:
         - Section 2: `{% include "core/services/_stats_row.html" %}`
         - Section 3: `{% include "core/services/_ci_pipeline_card.html" %}`
         - Section 4: `{% include "core/services/_recent_builds.html" %}`
