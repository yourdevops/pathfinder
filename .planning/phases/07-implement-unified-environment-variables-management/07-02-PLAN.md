**`_env_var_container.html`** — The outer container that groups vars by source level.

Structure:

- Lock icon on LEFT side with three states:
  - `var.locked_by == current_level` (locked here): calm red (`text-red-400`), clickable to toggle lock via `hx-post`
  - `var.locked_by` and locked_by != current_level (locked upstream): grey (`text-gray-600`), not clickable, `title="Locked on {{ var.locked_by }} level"`
  - No locked_by (unlocked): ghost icon (`text-gray-700/30 hover:text-gray-500`), clickable to toggle lock
  - Lock toggle only available at project and service levels (not environment)
- Key with tooltip: use Alpine.js `x-data="{ tip: false }"` with `@mouseenter="tip = true"` `@mouseleave="tip = false"`. Tooltip text: "From: {{ var.source }}{% if var.description %}. {{ var.description }}{% endif %}"
- Small info/pencil icon next to key for description editing (opens inline edit via hx-get)
- `=` separator
- Value: if empty, show `<span class="text-dark-muted italic text-sm">not set</span>`. If show_empty_warning and value is empty, add amber background: `bg-amber-900/10 border border-amber-500/30` on the row
- Edit/Delete buttons on the right (only if is_editable and source == current_level)
- Row div has `id="env-var-row-{{ var.key }}"`, `hx-target="this"`, `hx-swap="outerHTML"`
- Row styling: `flex items-center gap-3 px-4 py-2.5 rounded-lg bg-dark-surface/50 hover:bg-dark-surface`

**`_env_var_row_edit.html`** — Inline edit mode for a single variable.

- Hidden fields: `target_type`, `target_id`, `project_name`, `editing_key`

- Save button (submit), Cancel button (hx-get to fetch display row, reverting changes)

Same as `_env_var_row_edit.html` but with empty key/value, no editing_key, key field is editable.

Create `core/views/env_vars.py` with entity-specific HTMX endpoints. Per research recommendation, use entity-specific URL patterns (not a unified endpoint) because URLs encode permission context.

**Helper: `_get_target_object(request, target_type, target_id, project_name)`**
Resolves the target entity (Project, Service, or Environment) from request parameters. Returns (entity, project) tuple. Checks permissions using existing mixins/helpers.

**Views to create:**

1. **`EnvVarEditRowView`** (GET) — Returns `_env_var_row_edit.html` for a given variable key.
   - Parameters: target_type, target_id, project_name, key (query param)
   - Finds the var in the entity's env_vars list, returns edit partial
   - If key not found, returns 404

2. **`EnvVarDisplayRowView`** (GET) — Returns `_env_var_row.html` for a given variable key (used by cancel button).
   - Same parameters as EditRowView
   - Uses `resolve_env_vars()` from `core/utils.py` to get the resolved var with source/locked_by info
   - Returns display partial

3. **`EnvVarSaveView`** (POST) — Saves a variable (create or update).
   - Parameters from form: key, value, target_type, target_id, project_name, editing_key (optional)
   - Validates: key matches `^[A-Z][A-Z0-9_]*$`, key not a PTF_* prefix (system vars), locked upstream vars cannot be overridden
   - If editing_key provided: update existing var. If not: create new var.
   - Saves to entity's env_vars JSONField
   - Returns the display row partial for the saved var (HTMX swaps it in)

4. **`EnvVarDeleteView`** (DELETE or POST with _method) — Deletes a variable.
   - Validates: cannot delete system vars, cannot delete upstream vars
   - Removes var from entity's env_vars JSONField
   - Returns empty string (HTMX removes the row)

5. **`EnvVarAddRowView`** (GET) — Returns `_env_var_add_row.html` (empty row for adding).
   - Returns the add row partial

6. **`EnvVarToggleLockView`** (POST) — Toggles lock state on a variable.
   - Validates: only project and service levels support locking, cannot lock empty values, cannot toggle system var locks
   - Toggles lock boolean on the var
   - Returns display row partial with updated state

All views use `LoginRequiredMixin`. Permission checking:
- Project env vars: require ProjectOwnerMixin equivalent (check project membership with owner role)
- Service env vars: require ProjectContributorMixin equivalent
- Environment env vars: require ProjectContributorMixin equivalent
