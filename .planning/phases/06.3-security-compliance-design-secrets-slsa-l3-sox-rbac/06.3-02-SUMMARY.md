## Decisions Made
- Cosign 3.x with `--type slsaprovenance1` for SLSA v1.0 format -- the deprecated `slsaprovenance` (without suffix) generates v0.2
- Rekor transparency log disabled by default (`--tlog-upload=false`) -- enterprise privacy requirement
- CycloneDX JSON recommended for SBOM over SPDX -- better security/vulnerability focus, native VEX support, Cosign native type
- Two verification points only (build ingestion + deploy time) -- no separate promotion boundary verification to avoid operational complexity
- Trust steps repository via branch protection rather than signing individual CI steps -- complexity vs security tradeoff
- Minimum timestamp-authority >= 1.2.8 for DigiCert TSA certificate chain compatibility (sigstore/cosign#3632)
