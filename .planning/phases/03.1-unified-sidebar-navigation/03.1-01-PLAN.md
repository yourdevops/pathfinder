<task type="auto">
  <name>Task 3: Restructure nav.html with expandable sections using Persist</name>
  <files>core/templates/core/components/nav.html</files>
  <action>
Completely rewrite nav.html with the new expandable section structure:

1. Wrap entire nav content in single Alpine x-data with $persist for each section:
   ```html
   <div x-data="{
       serviceCatalogOpen: $persist(true).as('nav_serviceCatalog'),
       blueprintsOpen: $persist(false).as('nav_blueprints'),
       integrationsOpen: $persist({% if 'connections' in request.path or 'plugins' in request.path %}true{% else %}false{% endif %}).as('nav_integrations'),
       settingsOpen: $persist({% if 'settings' in request.path %}true{% else %}false{% endif %}).as('nav_settings')
   }">
   ```

2. Structure each section with the expandable pattern:

   **Service Catalog** (all users):
   - Parent: click toggles serviceCatalogOpen
   - Children: Services ({% url 'services:list' %}), Projects ({% url 'projects:list' %})
   - Highlight: active if 'services' or 'projects' in request.path
   - Icon: folder-open or similar

   **Blueprints** (all users):
   - Parent: click toggles blueprintsOpen
   - Children: Services ({% url 'blueprints:list' %}), Resources ({% url 'resources:list' %})
   - Highlight: active if 'blueprints' or 'resources' in request.path
   - Icon: template/document icon
   - Note: "Services" here refers to blueprint services (template catalog), not deployed services

   **Integrations** (all users):
   - Keep existing pattern but use $persist
   - Children: Connections ({% url 'connections:list' %}), Plugins ({% url 'connections:plugins' %})
   - Highlight: existing logic
   - Icon: link/connection icon (existing)

   **Settings** (admin only, {% if is_admin %}):
   - Parent: click toggles settingsOpen
   - Children: General, Users & Groups, Audit Log, Notifications
   - URLs: Use EXISTING settings namespace views (already in core/urls.py settings_patterns):
     - settings:general (GeneralSettingsView)
     - settings:user_management (UserManagementView)
     - settings:audit_logs (AuditLogsSettingsView)
     - settings:notifications (NotificationsView)
   - Highlight: use active_settings_section from context
   - Icon: cog icon (existing)
   - Note: No new placeholder views needed - all settings views already exist

3. Remove Documentation section (external link) - it was a placeholder

4. Keep the user section at bottom (existing pattern)

5. Use consistent styling for all sections:
   - Parent row: full width button, icon left, text center, chevron right

   - Active highlighting: bg-dark-border for active item
   - x-transition on the children container for smooth expand/collapse
  </action>
