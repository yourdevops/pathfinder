# Phase 3.1: Unified Sidebar Navigation - Context

**Gathered:** 2026-01-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Restructure the sidebar navigation to have expandable sections (Service Catalog, Blueprints, Integrations, Settings) with consistent behavior. Add a home/dashboard landing page. Implement context-aware navigation when entering project views. Remove breadcrumbs in favor of sidebar-based location awareness.

</domain>

<decisions>
## Implementation Decisions

### Home Dashboard
- Home is NOT a sidebar item — it's the default landing page
- Accessed by: (1) first visit to portal, (2) clicking DevSSP logo/text at sidebar top
- Hero section with prominent "+ Create Service" card/banner and description
- Stats cards showing quick counts: services, active deployments, recent builds
- Activity feed showing team activity (all activity across projects user has access to)
- Activity types: Services & Deployments focus (service created, deployed, build completed/failed)

### Expansion Behavior
- Multi-open: sections expand/collapse independently (not accordion)
- State persists across sessions via localStorage
- Entire row is click target for expand/collapse
- Active page item highlighted within its expanded section

### Context Switching
- Entering project context replaces sidebar entirely with project-specific nav
- Back button at top replaces DevSSP logo/text when in project context
- Project header shows project name/icon prominently at top of sidebar
- Project nav items: Details (first, opens by default), Services, Environments, Members
- Settings accessible from Details page, not a separate nav item

### Navigation Structure (Main)
- Service Catalog section: Services, Projects
- Blueprints section: Services (Phase 4 placeholder), Resources (placeholder)
- Integrations section: Connections, Plugins (existing)
- Settings section: General (placeholder), Users & Groups, Audit Log, Notifications (placeholder)

### Breadcrumbs
- No breadcrumbs anywhere in the application
- Sidebar is the source of truth for current location
- Back button on sidebar handles parent navigation

### Claude's Discretion
- Exact animation/transition for expand/collapse
- Icon choices for sections
- Stats card layout and design
- Activity feed item density and truncation
- Exact placement of back button vs project header

</decisions>

<specifics>

- Details page in project context should be the default landing, with settings configurable from there
- Start implementation from "Integrations" section, as it already has some of the functionality implemented.
