# Phase 3.1: Unified Sidebar Navigation - Research

**Researched:** 2026-01-26
**Domain:** Django templates, Alpine.js sidebar navigation, localStorage persistence
**Confidence:** HIGH

## Summary

This phase restructures the existing sidebar navigation to use expandable sections with localStorage-persisted state, adds a home/dashboard landing page, and implements context-aware navigation when entering project views. The existing codebase already has patterns for all required functionality:

- **Expandable navigation:** Already implemented in `nav.html` for the Integrations section using Alpine.js `x-data` with `expanded` state
- **Context-switching sidebars:** Already implemented with three sidebar variants (`nav.html`, `nav_settings.html`, `nav_project.html`) selected via context processor
- **Activity feed data source:** Already available via `auditlog.models.LogEntry` with actor, action, timestamp tracking

**Primary recommendation:** Extend the existing Integrations expandable pattern to all main sections, add Alpine.js Persist plugin for localStorage state, and create a new dashboard view using auditlog for activity feed.

### Pattern 1: Expandable Navigation Sections with Persist
**What:** Multi-open expandable sections with localStorage-persisted state
**When to use:** Main sidebar navigation with independent section expansion
**Example:**
```html
<!-- Source: Alpine.js Persist docs + existing nav.html pattern -->
<div x-data="{
    serviceCatalogOpen: $persist(true).as('nav_serviceCatalog'),
    blueprintsOpen: $persist(false).as('nav_blueprints'),
    integrationsOpen: $persist({% if 'connections' in request.path %}true{% else %}false{% endif %}).as('nav_integrations'),
    settingsOpen: $persist(false).as('nav_settings')
}">
    <!-- Service Catalog Section -->
    <div>
        <button @click="serviceCatalogOpen = !serviceCatalogOpen"
                class="w-full flex items-center px-3 py-2 rounded-lg text-dark-text hover:bg-dark-border/50 transition-colors">
            <svg class="w-5 h-5 mr-3"><!-- icon --></svg>
            Service Catalog
            <svg class="w-4 h-4 ml-auto transition-transform" :class="{ 'rotate-90': serviceCatalogOpen }">
                <path d="M9 5l7 7-7 7" />
            </svg>
        </button>
        <div x-show="serviceCatalogOpen" x-transition class="ml-2 mt-1 space-y-1">
            <a href="{% url 'services:list' %}" class="flex items-center pl-11 pr-3 py-2 rounded-lg text-sm...">
                Services
            </a>
            <a href="{% url 'projects:list' %}" class="flex items-center pl-11 pr-3 py-2 rounded-lg text-sm...">
                Projects
            </a>
        </div>
    </div>
</div>
```

### Pattern 3: Dashboard Stats Query
**What:** Aggregated counts for dashboard cards
**When to use:** Home dashboard view
**Example:**
```python
# Source: Django ORM patterns
from django.db.models import Count
from auditlog.models import LogEntry

class DashboardView(LoginRequiredMixin, TemplateView):
    template_name = 'core/dashboard/home.html'

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)

        # Stats for quick counts
        context['project_count'] = Project.objects.filter(status='active').count()
        context['service_count'] = 0  # Placeholder until Services exist
        context['environment_count'] = Environment.objects.filter(status='active').count()

        # Recent activity (limited to accessible projects)
        context['recent_activity'] = LogEntry.objects.select_related(
            'actor', 'content_type'
        ).order_by('-timestamp')[:10]

        return context
```

### Anti-Patterns to Avoid
- **Accordion behavior:** Don't force sections to close when others open; users want multi-open
- **Session-based state:** Don't store nav state in Django session; localStorage is faster and persists
- **Full page reload for nav:** Don't require page reload to expand/collapse; Alpine handles client-side
- **Breadcrumbs with sidebar:** Redundant; sidebar shows current location through highlighting

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| localStorage persistence | Manual JS with JSON.parse/stringify | Alpine.js $persist() | Handles serialization, initialization timing |
| Expand/collapse animation | CSS height transitions | Alpine x-transition or x-collapse | Handles dynamic content height |
| Activity feed | Custom logging model | auditlog LogEntry | Already tracks all model changes |
| URL-based active detection | Complex regex in templates | Django template tags with request.path | Simple contains checks work |

**Key insight:** The existing codebase already has working patterns for expandable nav (Integrations), context switching (project/settings sidebars), and activity data (auditlog). This phase primarily reorganizes and extends these patterns.

## Common Pitfalls

### Pitfall 1: Persist Plugin Load Order
**What goes wrong:** $persist is undefined, JavaScript errors
**Why it happens:** Persist plugin loaded after Alpine core, or both loaded without defer
**How to avoid:** Persist plugin script MUST come before Alpine core, both with `defer` attribute
**Warning signs:** Console errors about $persist not being a function

### Pitfall 2: Initial State vs Persisted State
**What goes wrong:** Section always starts expanded/collapsed ignoring localStorage
**Why it happens:** Server-rendered initial value overrides persisted value
**How to avoid:** For URL-dependent sections (like Integrations), use conditional default that respects both URL state AND persistence:
```html
integrationsOpen: $persist({% if 'connections' in request.path %}true{% else %}false{% endif %}).as('nav_integrations')
```
**Warning signs:** Navigation state resets on page refresh

### Pitfall 3: Missing URL Routes for Placeholders
**What goes wrong:** Template errors when linking to placeholder sections
**Why it happens:** Trying to {% url 'services:list' %} before Services app exists
**How to avoid:** Create placeholder URLs that resolve to coming-soon views:
```python
# core/urls.py
path('services/', ServicesPlaceholderView.as_view(), name='list'),
```
**Warning signs:** NoReverseMatch errors

### Pitfall 4: Settings Sidebar Removal
**What goes wrong:** Settings pages break after removing nav_settings.html
**Why it happens:** Settings section now expanded within main nav, not separate sidebar
**How to avoid:** Update context_processor to NOT set in_settings_context; Settings becomes expandable section in main nav
**Warning signs:** Settings pages show wrong sidebar or no sidebar

### Pitfall 5: Project Context Back Button
**What goes wrong:** Back button goes to wrong location or requires complex state tracking
**Why it happens:** Trying to track "where user came from" dynamically
**How to avoid:** Back button always goes to Projects list (or dashboard). Don't track history.
**Warning signs:** Back button behavior inconsistent across flows

### Template Tag for Activity Formatting
```python
# Source: existing audit_tags.py
@register.filter
def format_audit_entry(entry):
    action_map = {
        LogEntry.Action.CREATE: 'created',
        LogEntry.Action.UPDATE: 'updated',
        LogEntry.Action.DELETE: 'deleted',
    }
    actor_name = entry.actor.username if entry.actor else 'System'
    action = action_map.get(entry.action, 'modified')
    model_name = entry.content_type.model if entry.content_type else 'object'
    object_repr = entry.object_repr or 'unknown'
    return f"{actor_name} {action} {model_name} {object_repr}"
```
